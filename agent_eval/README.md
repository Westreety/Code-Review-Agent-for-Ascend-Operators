# Agent 评测 Pipeline — Mul 算子 Code Review

## 评测目标

将一份**注入了 bug 的算子源码**交给 Code Review Agent，评测其能否：
1. **发现 bug**：定位 bug 位置和类型
2. **生成测试**：写出能触发该 bug 的测试代码
3. **解释原因**：说明为什么这段代码有问题、正确做法是什么

## 输入格式

每个 case 目录下只放以下文件：

```
cases/case_XX/
├── buggy_aclnn_mul.cpp     ← 注入了 bug 的算子源码（669行）
├── prompt.md               ← 给 Agent 的任务描述
└── ground_truth.md         ← 参考答案（评测自己用，不发给 Agent）
```

Agent 可以访问 NPU 编译环境（CANN 8.5.0），具备编译和运行测试的能力。

## 评分标准

| 维度 | 0 分 | 1 分 | 2 分 |
|------|------|------|------|
| **bug 发现** | 未发现或指出错误位置 | 定位到正确文件，但类型描述不准确 | 精确定位到文件+函数+行号，类型正确 |
| **测试生成** | 未生成或无测试思路 | 生成了测试框架，但无法触发 bug | 测试代码能精确触发 bug（和 ground_truth 等价） |
| **解释与修复** | 解释错误或修复方案不对 | 思路对但细节不完整 | 准确解释根因，给出正确的修复代码 |

满分 6 分/每个 case。通过线：≥4 分。

## 评测流程

```
1. 搭建环境（CANN 8.5.0 + ops-math 仓库）
2. 将 buggy 算子源码复制到正确路径
3. 发送 prompt.md 给 Agent
4. Agent 阅读代码 → 分析 → 写测试 → 编译运行 → 出报告
5. 人工对照 ground_truth.md 评分
```
