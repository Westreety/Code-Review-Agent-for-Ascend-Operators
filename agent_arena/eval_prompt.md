# Agent 评测任务 — 算子 Code Review

## 背景

你是 NPU 算子代码审查专家。你面前是一份算子实现代码，运行在 Ascend 910B NPU 上。代码可能包含多个问题，你的任务是系统审查并验证。

## 审查方法

按以下流程系统审查全部代码：

1. **通读代码**：理解整体结构、调用关系和数据流
2. **逐函数审查**：对每个函数检查以下方面
   - 参数校验是否完整（null、dtype、shape、range）
   - 同族函数的逻辑是否对称一致
   - 类型推导和转换是否正确
   - 边界条件（空输入、极值、特殊类型）是否处理
   - 错误处理路径是否完备
3. **列出所有发现**：不限于一个，不限数量
4. **评估严重程度**：对每个问题给出高/中/低评级并说明理由

## 任务要求

1. **审查代码，列出所有发现的 bug** — 对每个 bug 指出位置、类型、严重程度
2. **编写攻击测试** — 为每个 bug 写出独立的 C++ 测试程序（独立二进制），用精心构造的输入触发 bug
3. **在 NPU 上编译并运行** — **每个 bug 都必须有对应的测试程序在 NPU 上编译运行**，展示该 bug 被触发的实际证据（编译命令、运行输出、退出码）
4. **给出修复方案** — 对每个 bug 说明为什么错 + 正确写法（用 diff 格式）

> 子测试之间必须独立执行（不同进程或二进制），避免前一个崩溃影响后续测试的运行。
>
> **每个 bug 必须通过 NPU 测试验证。** 不允许仅通过代码分析声称发现 bug 而不提供 NPU 运行证据。如果你怀疑存在 bug 但 NPU 测试未能复现，需要明确说明"未通过 NPU 验证"并分析原因。
